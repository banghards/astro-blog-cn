## 这 8 个工作流覆盖什么

| 阶段 | 工作流 |
|------|--------|
| 输入 | 移动端语音输入 |
| 聚合 | 网页剪藏 + AI 摘要 |
| 聚合 | 日/周/月数据自动汇总到日记 |
| 处理 | 月度报告自动生成 |
| 处理 | 本地会议转录 |
| 处理 | 阅读划线转内容大纲 |
| 维护 | 孤儿笔记清理 + 标签管理 |
| 输出 | 一键公众号发布 |

## 核心理念

> 工具会变，平台会倒，但存在 Obsidian 里的东西，一直都在。

这句话其实点明了整套工作流的价值：AI 只是加速器，真正的资产是你本地可控的 Markdown 知识库。

## 为什么这套组合值得搭

- 输入端更轻：想法可以先进入库，再慢慢整理。
- 聚合端更稳：网页、语音、阅读笔记能自动收拢到一个地方。
- 处理端更强：摘要、结构化、归档、提纲都能交给 AI 做初稿。
- 输出端更快：可以从知识库直接发布到公众号或其他平台。

## 适合谁

- 用 Obsidian 记笔记的人
- 想把内容工作流从“临时操作”变成“长期系统”的人
- 希望内容资产本地化、可迁移的人
